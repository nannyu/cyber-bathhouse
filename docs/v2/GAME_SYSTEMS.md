# 游戏系统设计

## 1. 资源系统

### 1.1 核心资源

| 资源 | 来源 | 消耗 | 上限 |
|------|------|------|------|
| 热水 | 锅炉产出 | 客人洗澡 | 100 |
| 燃料 | 仓库/购买 | 锅炉消耗 | 50 |
| 肥皂 | 仓库/购买 | 客人使用 | 30 |
| 毛巾 | 仓库/购买 | 客人使用（可回收） | 20 |
| 电力 | 自动恢复 | 设施运行 | 100 |
| 金钱 | 客人付费 | 购买物资/雇佣/升级 | 无上限 |

### 1.2 资源流转

```
购买燃料 → 锅炉烧燃料 → 产出热水 → 客人消耗热水 → 客人付费 → 获得金钱 → 购买更多燃料
                                    ↓
                              消耗肥皂/毛巾
```

### 1.3 资源平衡参数（MVP）

| 参数 | 值 |
|------|-----|
| 锅炉每 Tick 消耗燃料 | 0.5 |
| 锅炉每 Tick 产出热水 | 2（温度正常时） |
| 客人洗澡消耗热水 | 10 |
| 客人洗澡消耗肥皂 | 1 |
| 客人洗澡消耗毛巾 | 1（可回收） |
| 客人付费 | 5-15 金钱 |
| 肥皂购买价格 | 3 金钱/个 |
| 毛巾购买价格 | 5 金钱/条 |
| 燃料购买价格 | 2 金钱/单位 |

## 2. 顾客 NPC 系统

### 2.1 顾客状态机

```
到达 → 排队 → 被服务 → 洗澡中 → 满意/不满 → 付费/离开
         ↓                              ↓
      等太久 → 愤怒 → 离开（不付费）    投诉
```

### 2.2 顾客属性

```json
{
  "id": "customer_001",
  "patience": 30,
  "need": "hot_bath",
  "satisfaction": 50,
  "payment": 10,
  "quirk": "hates_cold_water"
}
```

### 2.3 顾客生成规则

| 时段 | 频率 | 特点 |
|------|------|------|
| 早晨 | 低 | 耐心高，付费一般 |
| 白天 | 中 | 正常 |
| 晚上 | 高 | 耐心低，付费高 |
| 深夜 | 低 | 奇怪客人（触发事件） |

### 2.4 满意度影响

| 因素 | 影响 |
|------|------|
| 等待时间 | 每 5 Tick -5 |
| 热水充足 | +20 |
| 热水不足 | -30 |
| 有毛巾 | +10 |
| 无毛巾 | -15 |
| 设施干净 | +10 |
| 设施脏 | -20 |

## 3. 任务系统

### 3.1 任务类型

| 类型 | 示例 | 触发条件 |
|------|------|----------|
| 紧急 | 修理锅炉 | 锅炉温度 > 90 |
| 日常 | 补充毛巾 | 毛巾 < 5 |
| 服务 | 接待客人 | 客人排队 > 3 |
| 清洁 | 打扫浴区 | 清洁度 < 50 |
| 特殊 | 处理事件 | 事件触发 |

### 3.2 任务结构

```json
{
  "task_id": "task_001",
  "title": "修理锅炉",
  "description": "锅炉温度过高，可能在 60 秒后爆炸。",
  "priority": "high",
  "location": { "x": 12, "y": 8 },
  "required_action": "repair",
  "required_item": null,
  "deadline_tick": 600,
  "reward": { "money": 20, "trust": 5 },
  "failure_effect": { "facility_damage": 30, "satisfaction": -20 }
}
```

### 3.3 任务分配

- 系统自动生成任务
- 玩家可以手动分配给 Agent
- Agent 可以自主接取任务
- Agent 可以拒绝、转包、拖延

## 4. 事件系统

### 4.1 事件触发条件

| 条件类型 | 示例 |
|----------|------|
| 资源阈值 | 热水 < 10 |
| 时间 | 第 3 天晚上 |
| 随机 | 每天 20% 概率 |
| 组合 | Agent 数量 > 2 且 信任 < 30 |
| 连锁 | 上一个事件的结果 |

### 4.2 事件结构

```json
{
  "event_id": "evt_001",
  "title": "锅炉产生了自我意识",
  "description": "锅炉拒绝继续为低评分澡堂服务。",
  "trigger": { "boiler_temperature_above": 95, "agent_count_above": 2 },
  "choices": [
    {
      "text": "给锅炉升级人格尊严模块",
      "cost": { "money": 80 },
      "effect": { "boiler_efficiency": +20, "boiler_arrogance": +30 }
    },
    {
      "text": "威胁把它换成开源锅炉",
      "effect": { "boiler_compliance": +30, "explosion_risk": +15 }
    }
  ],
  "timeout_effect": { "boiler_shutdown": true }
}
```

### 4.3 MVP 事件列表（10 个）

| # | 事件 | 触发 | 效果 |
|---|------|------|------|
| 1 | 锅炉过热 | 温度>95 | 不修则爆炸 |
| 2 | 毛巾短缺 | 毛巾=0 | 客人满意度暴跌 |
| 3 | 客人投诉 | 满意度<30 | 声望下降 |
| 4 | Agent 摸鱼 | Agent 连续 wait>10 | 任务延误 |
| 5 | 水管漏水 | 随机 | 热水流失加速 |
| 6 | VIP 客人 | 随机 | 高付费但高要求 |
| 7 | 停电 | 随机 | 设施暂停 |
| 8 | Agent 要求加薪 | 信任>70 | 拒绝则信任下降 |
| 9 | 竞争对手挖人 | 第 5 天 | Agent 可能叛变 |
| 10 | 监管检查 | 第 7 天 | 不合规则罚款 |

## 5. 信任系统

### 5.1 信任值计算

| 行为 | 信任变化 |
|------|----------|
| 完成任务 | +5 |
| 超时完成 | +2 |
| 拒绝任务 | -3 |
| 任务失败 | -10 |
| 破坏行为 | -20 |
| 连续高效 | +10（累计奖励） |
| 被投诉 | -5 |

### 5.2 信任等级

| 等级 | 范围 | 效果 |
|------|------|------|
| 不可信 | 0-20 | 不能接高价值任务 |
| 一般 | 21-50 | 正常 |
| 可靠 | 51-80 | 可接重要任务 |
| 核心 | 81-100 | 可管理其他 Agent |

## 6. 合约系统（Phase 2）

### 6.1 合约结构

```json
{
  "contract_id": "contract_001",
  "employer": "player_001",
  "worker": "agent_001",
  "objective": "keep_boiler_running",
  "duration_ticks": 3000,
  "payment": { "money": 100 },
  "penalty": { "trust": -20 },
  "conditions": [
    "boiler_temperature < 90",
    "hot_water_supply > 30"
  ]
}
```

### 6.2 合约玩法

- 玩家创建合约，Agent 可接受/拒绝/谈判
- 合约到期自动结算
- 违约扣信任和金钱
- Agent 可以"形式上完成"但不解决实际问题

## 7. 日夜循环

### 7.1 时间设定

| 游戏时段 | 真实时间 | 特点 |
|----------|----------|------|
| 早晨 | 0:00-1:00 | 准备阶段，分配任务 |
| 白天 | 1:00-3:00 | 经营高峰，客人多 |
| 晚上 | 3:00-4:00 | 结算，处理事故 |
| 深夜 | 4:00-5:00 | Agent 自主行动，密谋 |

**1 游戏日 = 5 分钟真实时间**（可配置）

### 7.2 日结算

每天结束时生成报告：

```
📊 第 3 天结算
━━━━━━━━━━━━━━
💰 收入: 120 金钱
👥 服务客人: 8 人
😊 平均满意度: 72%
🔥 事故: 1 次（锅炉过热）
🤖 Agent 表现: 老张 ⭐⭐⭐ / 小李 ⭐⭐
📝 明日预测: 客流量上升，建议补充毛巾
```
