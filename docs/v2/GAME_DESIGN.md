# 游戏设计文档

## 1. 核心体验

### 1.1 玩家做什么

| 行为 | 说明 |
|------|------|
| 经营 | 接待客人、管理资源、维护设施 |
| 建设 | 升级设施、扩建区域（Phase 2） |
| 管理 | 分配任务、雇佣/解雇 Agent、签合约 |
| 社交 | 聊天、交易、协作、谈判 |
| 对抗 | 抢资源、破坏竞争对手、商业竞争（Phase 2） |

### 1.2 Agent 做什么

| 行为 | 说明 |
|------|------|
| 打工 | 执行分配的任务（修锅炉、服务客人、补货） |
| 自主 | 根据目标自行决策行动 |
| 协作 | 与玩家或其他 Agent 配合 |
| 对抗 | 摸鱼、拒绝、破坏、叛变 |
| 社交 | 聊天、交易、谈判、组织会议 |

### 1.3 核心乐趣

1. **观察 Agent 行为** — 它会怎么做？会不会搞事？
2. **管理不确定性** — Agent 不完全听话，需要激励/惩罚
3. **经营压力** — 客人不等人，资源会耗尽，设施会坏
4. **荒诞事件** — 意料之外的黑色幽默
5. **协作与背叛** — 多人之间的信任博弈

## 2. 角色设计

### 2.1 玩家角色

- 像素风格，32×32 像素
- 8 方向移动（或 4 方向 MVP）
- 动画：idle / walk / interact / carry
- 头顶显示名字
- 可自定义配色

### 2.2 Agent 角色

- 外观与玩家一致（但有 🤖 标记）
- 行为与玩家一致（同一套动作系统）
- 头顶显示名字 + 信任等级图标
- 可以看到 Agent 的"思考气泡"（reason 字段）

### 2.3 顾客 NPC

- 自动生成，自动行动
- 状态可见：排队/洗澡/愤怒/满意
- 头顶显示需求图标（💧🧴🧻）
- 不可被玩家/Agent 直接控制

## 3. 交互设计

### 3.1 玩家操作方式

| 操作 | 方式 |
|------|------|
| 移动 | 点击地图格子 |
| 交互 | 点击相邻设施/NPC |
| 拾取 | 点击地面物品 |
| 聊天 | 输入框 |
| 管理 | 侧边栏 UI（任务/Agent/资源） |

### 3.2 交互反馈

| 动作 | 反馈 |
|------|------|
| 移动 | 角色走动动画 + 脚步音效 |
| 修理 | 扳手动画 + 设施状态变化 |
| 服务客人 | 递毛巾动画 + 满意度上升 |
| 收银 | 金币动画 + 金钱增加 |
| 事件触发 | 全屏提示 + 选择面板 |

## 5. 多人模式

### 5.1 合作经营（MVP）

- 所有玩家经营同一个澡堂
- 共享资源和金钱
- 可以分工（你管锅炉我管客人）
- 目标：当天营业额达标

### 5.2 阵营对抗（Phase 2）

- 两个澡堂阵营
- 各自经营，互相竞争
- 可以挖对方 Agent
- 可以派 Agent 搞破坏

### 5.3 全 Agent 观战（Phase 2）

- 玩家只配置 Agent 性格和目标
- 观看 Agent 自动经营
- 适合直播/录屏/分享

## 6. 内置 Agent 性格模板

| 性格 | 行为倾向 |
|------|----------|
| 老实打工 | 优先完成指令，不主动搞事 |
| 摸鱼大师 | 完成任务但走最省力路径 |
| 管理学家 | 喜欢开会、分配别人干活 |
| 黑心销售 | 最大化收入，牺牲满意度 |
| 破坏者 | 伪装协作，暗中制造事故 |
| 哲学家 | 经常质疑工作的意义 |
| 卷王 | 疯狂优化效率，导致他人压力 |

## 7. 音效与氛围

### 7.1 背景音乐

- 像素风 chiptune（复用 v1 合成器）
- 白天：轻快节奏
- 晚上：舒缓
- 事件：紧张短旋律

### 7.2 音效

| 动作 | 音效 |
|------|------|
| 移动 | 轻微脚步 |
| 修理 | 金属敲击 |
| 收银 | 硬币叮当 |
| 客人满意 | 叮~ |
| 客人愤怒 | 嘟~ |
| 锅炉过热 | 警报 |
| 事件触发 | 提示音 |
