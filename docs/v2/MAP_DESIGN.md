# 地图设计

## 1. 地图系统

### 1.1 Tile-based 网格

- 格子大小：32×32 像素
- 地图尺寸（MVP）：20×13 格（640×416 像素）
- 使用 Tiled Map Editor 编辑，导出 JSON 格式
- 支持多图层：地面层、物件层、碰撞层

### 1.2 图层结构

| 图层 | 说明 |
|------|------|
| ground | 地面瓷砖/木板/水面 |
| objects | 设施、家具、装饰 |
| collision | 不可通行区域 |
| zones | 功能区域标记（大堂/浴区/仓库/锅炉房） |

## 3. 功能区域

### 3.1 大堂 (Lobby)

- 位置：左上区域 (0,0)-(9,4)
- 功能：客人入口、排队等候、收银结账
- 设施：等候椅×3、收银台×1、入口门×1
- 交互：`collect`（收费）、`serve`（引导客人）

### 3.2 浴区 (Bath)

- 位置：左下区域 (0,5)-(9,11)
- 功能：洗澡服务核心区
- 设施：浴池×4格、淋浴×4格、休息椅×2
- 交互：`serve`（服务客人）、`clean`（清洁）
- 资源消耗：热水、肥皂

### 3.3 仓库 (Storage)

- 位置：右上区域 (10,0)-(19,4)
- 功能：物资存储和补给
- 设施：肥皂架×2、毛巾架×2
- 交互：`pickup`（取物资）、`restock`（补货）

### 3.4 锅炉房 (Boiler Room)

- 位置：右下区域 (10,5)-(19,11)
- 功能：热水生产、设施维护
- 设施：锅炉×2、控制台×1、工具箱×1
- 交互：`fuel`（加燃料）、`repair`（修理）
- 风险：过热爆炸

## 4. 设施详细设计

### 4.1 锅炉 (Boiler)

```json
{
  "type": "boiler",
  "state": {
    "temperature": 75,
    "fuel": 60,
    "condition": 85,
    "hot_water_output": 10
  },
  "thresholds": {
    "overheat": 95,
    "explosion": 100,
    "low_fuel": 20,
    "breakdown": 30
  }
}
```

- 每 Tick 消耗燃料，产出热水
- 温度过高需要修理
- 燃料耗尽停止产热水
- 状况低于 30 会故障

### 4.2 浴池 (Bath)

```json
{
  "type": "bath",
  "state": {
    "cleanliness": 90,
    "hot_water_level": 70,
    "occupied_by": null
  }
}
```

- 客人使用后清洁度下降
- 需要定期清洁
- 热水不足时客人不满意

### 4.3 收银台 (Counter)

```json
{
  "type": "counter",
  "state": {
    "queue_length": 3,
    "money_collected": 0
  }
}
```

- 客人排队等待
- 需要角色操作收银
- 等待过久客人离开

---

## 5. 寻路

- 使用 A* 算法
- 碰撞层标记不可通行格子
- 角色每 Tick 移动 1 格
- `move_to` 动作自动计算路径
- 路径被阻挡时重新计算

---

## 6. 视野系统

- 每个角色视野半径：5 格
- Agent 只能感知视野内的实体
- 墙壁阻挡视线（可选，MVP 可不做）
- 公共信息（资源总量、时间）全局可见
